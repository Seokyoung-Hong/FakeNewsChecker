# FakeNewsChecker

이 저장소는 **API 연동을 처음 배워 보는 사람도 실습할 수 있는 Python 프로젝트**입니다.

지금 이 프로젝트는 웹 화면으로 실행은 되지만, 실제 뉴스 분석 API가 붙어 있는 상태는 아닙니다. 현재 분석 결과는 연습용으로 만들어진 스텁 결과입니다.

그래서 이 저장소를 처음 보는 사람은 이렇게 생각하면 됩니다.

> **기존 구조를 먼저 이해하려고 애쓰지 말고, 새 폴더에 Python 파일 하나를 만들어 API 호출부터 해 보면 된다.**

## 1. 가장 쉬운 시작 방법

저장소 안에 원하는 폴더를 새로 하나 만드세요.

예를 들면:

```text
practice/
└── my_first_api.py
```

또는

```text
beginner/
└── test_call.py
```

폴더 이름은 아무거나 괜찮습니다.

그 다음에는 그 Python 파일 안에서 아래만 해 보면 됩니다.

1. API 주소 적기
2. 요청 보내기
3. 응답 출력하기
4. JSON이면 값 꺼내 보기
5. 에러 나면 메시지 읽어 보기

## 2. 처음에는 이것만 해도 충분합니다

처음 API를 배울 때 중요한 것은 아래입니다.

- 요청이 나가는지
- 응답이 오는지
- JSON을 읽을 수 있는지
- 원하는 값만 꺼낼 수 있는지
- 실패 원인을 볼 수 있는지

반대로 처음부터 아래를 신경 쓸 필요는 없습니다.

- 프로젝트 구조
- 파일을 어느 폴더에 넣어야 하는지
- 기존 코드와 어떻게 연결해야 하는지
- 복잡한 내부 동작 방식

## 3. 이 저장소의 현재 상태

아주 쉽게 말하면 지금 상태는 이렇습니다.

- 웹 화면은 실행됩니다.
- URL을 넣으면 결과 화면도 나옵니다.
- 하지만 그 결과는 진짜 분석 결과가 아니라 연습용입니다.
- 아직 실제 외부 API를 붙인 상태는 아닙니다.

추가로 현재 프로토타입에는 비용 절감을 위한 간단한 캐시가 들어 있습니다.

- `CRAWLER_PROVIDER=hyperbrowser`일 때 같은 정규화 URL의 다운로드 결과는 **10일 동안 재사용**됩니다.
- Hive 이미지 분석도 같은 상위 3개 이미지 URL 조합이면 **10일 동안 재사용**됩니다.
- 로컬 모델 경로는 `GET /local-model`, `POST /local-model`이며 Ollama 기반 분석을 사용합니다.

즉, **실제 API 연동을 연습해 보기 좋은 상태**라고 생각하면 됩니다.

## 4. 실행해 보고 싶다면

### 시작

```bash
docker compose up --build
```

### 접속

- `http://localhost:8000/`

### 확인할 것

1. 홈 화면이 열리는지
2. URL을 넣고 결과 화면으로 넘어가는지
3. 결과가 화면에 보이는지
4. `/local-model`에서도 같은 흐름으로 결과가 나오는지

### 종료

```bash
docker compose down
```
