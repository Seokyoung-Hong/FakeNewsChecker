# 기여 가이드

이 문서는 **처음으로 API 연동을 배워 보는 사람**을 위한 안내입니다.

어렵게 생각하지 않아도 됩니다. 이 저장소의 기존 구조를 꼭 지킬 필요는 없습니다. 처음에는 `app/` 안을 건드리지 않아도 됩니다. **원하는 폴더를 새로 만들고, 그 안에 독립적인 Python 파일 하나를 만들어서 API 호출 연습부터 시작하면 됩니다.**

## 1. 먼저 알아둘 점

- 이 저장소는 지금도 실행은 됩니다.
- 하지만 현재 분석 기능은 진짜 뉴스 판별이 아니라 **연습용 스텁**입니다.
- 즉, 아직 실제 외부 API를 붙인 상태는 아닙니다.

그래서 처음 배우는 사람은 이렇게 생각하면 됩니다.

> "기존 코드를 완벽히 이해한 뒤 시작해야 하는 프로젝트"가 아니라,
> **새 파일 하나 만들어서 API 호출 연습을 해 봐도 되는 프로젝트**입니다.

## 2. 가장 쉬운 시작 방법

예를 들어 저장소 루트에 아래처럼 새 폴더를 만들어도 됩니다.

```text
practice/
└── my_first_api.py
```

또는 이런 식도 괜찮습니다.

```text
my_tests/
└── api_test.py
```

폴더 이름은 아무거나 괜찮습니다.

중요한 것은 다음입니다.

1. 새 폴더를 만든다.
2. Python 파일을 하나 만든다.
3. 그 파일에서 외부 API 호출을 해 본다.
4. 응답이 잘 오는지 출력해 본다.
5. 그다음 필요하면 프로젝트에 천천히 연결한다.

## 3. 처음에는 기존 구조를 신경 쓰지 마세요

처음 배우는 단계에서는 아래를 신경 쓰지 않아도 됩니다.

- 어느 폴더에 넣어야 하는지
- 서비스 파일인지 아닌지
- 라우터인지 아닌지
- 저장소 계층이 무엇인지
- 의존성 주입이 무엇인지

처음에는 **"Python으로 HTTP 요청 보내기"**, **"응답 JSON 보기"**, **"필요한 값 꺼내기"** 이 세 가지만 해도 충분합니다.

## 4. 추천 학습 순서

### 1단계 — 독립 파일에서 API 호출 연습

새 폴더와 새 파일을 만든 뒤, 외부 API 응답을 받아서 `print()` 해 보세요.

예시로 해 볼 수 있는 것:

- 요청 보내기
- 상태 코드 확인하기
- JSON 응답 출력하기
- 필요한 필드만 꺼내기
- 에러가 나면 예외 메시지 보기

### 2단계 — 입력값을 바꿔 보기

예를 들면 아래를 바꿔 가며 실험하면 됩니다.

- URL 값 바꾸기
- 요청 body 바꾸기
- 헤더 추가하기
- API 키 넣기
- 실패 케이스 만들기

### 3단계 — 결과를 파일로 저장해 보기

응답 결과를 화면에만 출력하지 말고 `.json` 파일이나 `.txt` 파일로 저장해 보세요.

이 단계까지 해 보면 “API 호출 → 응답 확인 → 결과 저장” 흐름을 익힐 수 있습니다.

### 4단계 — 필요하면 프로젝트에 연결하기

그 다음에야 기존 프로젝트 화면과 연결할지 고민하면 됩니다.

즉, **처음부터 프로젝트 구조 안에 예쁘게 넣으려고 하지 않아도 됩니다.**

## 5. 이 저장소에서 초보자에게 가장 중요한 목표

이 저장소에서 처음 배우는 사람에게 중요한 것은 아래입니다.

- API를 한 번 직접 호출해 보기
- 응답을 읽어 보기
- 원하는 데이터를 꺼내 보기
- 실패했을 때 오류를 확인해 보기
- 나중에 그 결과를 웹 화면에 붙일 수 있다는 감각 익히기

## 6. 연결은 나중에 해도 됩니다

처음부터 기존 `app/` 코드와 연결하려고 하면 어려울 수 있습니다.

그래서 아래 순서를 권장합니다.

1. 독립 파일에서 API 호출 성공
2. 독립 파일에서 응답 정리 성공
3. 독립 파일에서 저장 또는 출력 성공
4. 그다음 기존 프로젝트에 붙이기 시도

이 순서가 가장 부담이 적습니다.

## 7. 처음 시도할 때 체크할 것

- Python 파일이 실행되는지
- API 요청이 실제로 나가는지
- 응답이 오는지
- JSON 파싱이 되는지
- 에러가 나면 원인을 읽을 수 있는지

이 다섯 가지만 확인해도 첫 단계로 충분합니다.

## 8. 프로젝트에 나중에 연결하고 싶다면

나중에 정말 연결하고 싶어졌을 때만 기존 코드를 보면 됩니다.

그때 참고할 만한 파일은 아래 정도만 알면 충분합니다.

- `README.md`
- `docs/IMPLEMENTATION_GUIDE_ANALYSIS_PYTHON.md`
- `app/routers/analysis.py`

하지만 이것도 **처음부터 읽어야 하는 필수 코스는 아닙니다.**

## 9. 결론

이 저장소에서 처음 배우는 사람은 이렇게 시작하면 됩니다.

> 기존 구조를 지키려고 애쓰지 말고,
> **새 폴더를 하나 만들고 독립적인 Python 파일에서 API 호출부터 해 본다.**

그게 가장 좋은 시작입니다.
